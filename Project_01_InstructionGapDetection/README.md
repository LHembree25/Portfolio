# Instruction Gap Detection – DSC 670 Milestone 2

This project evaluates how GPT 4o mini identifies gaps, ambiguity, and missing steps in procedural instructions. The notebook contains five structured experiments that test direct gap detection, domain ambiguity, step-by-step audits, and comparative analysis.

## Tools Used
Python  
Jupyter Notebook  
OpenAI GPT 4o mini

## Notebook Contents
- Prompt design for five experiments  
- Model outputs for each experiment  
- Commentary evaluating clarity, consistency, and domain awareness  
- Assessment of whether prompting alone is sufficient for gap detection

## Key Results
The model consistently identified missing microsteps, unclear transitions, domain-specific assumptions, and ambiguous terminology. The outputs were stable across all five experiments. Prompt-based methods were sufficient for detecting instruction gaps without additional training.

## Why This Project Matters
This project demonstrates applied generative AI evaluation and technician-level prompt engineering. It supports future work in automated instruction auditing and workflow improvement.
